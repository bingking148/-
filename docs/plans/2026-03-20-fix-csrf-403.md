# 修复 CSRF 403 错误 - 创建会话失败

> **Goal:** 彻底解决前端调用 POST /api/sessions 时返回 403 Forbidden 的问题

**Architecture:** 通过正确配置 Django REST Framework 和 CORS，确保前后端分离架构下的 API 请求正常工作

**Tech Stack:** Django, Django REST Framework, React, CORS

---

## 问题分析

当前问题：
- GET 请求正常工作 (200)
- POST /api/sessions 返回 403 Forbidden
- 已尝试禁用 CSRF 中间件、添加 @csrf_exempt 装饰器均无效

根本原因：
Django REST Framework 的 SessionAuthentication 仍然在进行 CSRF 检查

## 解决方案

### Task 1: 修改 DRF 认证配置

**Files:**
- Modify: `easys_django/easys_django/settings.py`

**Step 1: 修改 REST_FRAMEWORK 配置**

将 SessionAuthentication 移除或调整顺序：

```python
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework.authentication.TokenAuthentication',
    ],
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.AllowAny',
    ],
    'DEFAULT_RENDERER_CLASSES': [
        'rest_framework.renderers.JSONRenderer',
    ],
}
```

**Step 2: 验证修改**

重启服务器后测试 POST 请求

---

### Task 2: 创建自定义认证类（备选方案）

如果 Task 1 无效，创建完全绕过 CSRF 的认证：

**Files:**
- Create: `easys_django/apps/api/authentication.py`

```python
from rest_framework.authentication import BaseAuthentication
from rest_framework import exceptions

class CsrfExemptSessionAuthentication(BaseAuthentication):
    """绕过 CSRF 检查的会话认证"""
    
    def authenticate(self, request):
        # 获取 Django 用户，但不检查 CSRF
        from django.contrib.auth.models import AnonymousUser
        
        if hasattr(request, 'user'):
            return (request.user, None)
        return (AnonymousUser(), None)
```

**Files:**
- Modify: `easys_django/easys_django/settings.py`

```python
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'apps.api.authentication.CsrfExemptSessionAuthentication',
    ],
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.AllowAny',
    ],
}
```

## 回滚计划

如果所有方案都失败：
1. 恢复原始 settings.py
2. 考虑使用 Django 的 @csrf_exempt 在视图层面处理
3. 或者前端获取 CSRF token 并添加到请求头
